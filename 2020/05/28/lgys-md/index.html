git小结

  • git的基本原理
    • 1： git 依靠哈希算法机制来从根本上保证数据的完整性
    • 2： git 把数据看成小型文件系统的一组快照，每次提交更新等操作，git都会产生一组索引和快照，工作方式成为快照流，因此查找效率高效
    • 3： git 分支操作，依赖于指针变化
  • git 命令操作
    • git clone ‘ssh://name’
      克隆远程仓库
    • git init
      创建本地git
    • touch name
      创建本地文件
    • git add .
      提交到本地缓存
    • git add -A
      将所有变动提交为本地缓存
    • git commit -m ‘message’
      将变动提交到本地仓库
    • git push origin branch
      提交到远程仓库分支
    • git remote add origin ‘ssh’
      添加远程仓库至本地
    • git log
      查看提交历史
    • git reset –herd ‘id’
      根据id回滚
    • git revert -n ‘id’
      删除中间某一次版本，保留后面的版本
    • git rebase -i HEAD~4
      整理四次提交为一个，避免代码污染，分支分叉
    • git branch -d ‘name’
      删除本地分支
    • git push origin –delete ‘name’
      删除远程分支
    • 还原删除的分支
      1 git log -g 查看删除哈希值
      2 git branch ‘name’ head@{1}

    es6 语法小结

  • let和const的作用域,如果用es5去实现,应该如何实现,或者说在es6转es5是,是如何处理let和const的–defineProterty的writeable
    let与const定义区域块变量，同一区域块中不能重复声明，var可以定义全局变量，可以提升声明的作用域。es5在函数中声明变量，更改作用域。let的defineProterty的writeable为true，可以修改，const的defineProterty的writeable为false，不能修改。

  • 你对Iterator迭代器的理解,与for of搭配使用是的调用原理–Symbol.iterator接的自动调用
    Iterator迭代器 ==> for of
    原理：迭代器具有next()方法，每次调用next方法时会返回一个key值与value值

  • 你知道的都有那哪些定义模块的规范.包的规范,amd,cmd,commonjs,esmoudle.cmd是如何实现包的就近引用的.

    • AMD
      AMD是requireJS倡导的一种模块化规范，推崇依赖前置；在requireJS中模块是通过define来进行定义的，如果模块之间相互依赖，需要先将依赖模块导入进来，待导入完毕之后，在通过回掉函数的方式执行后面的代码，有效的解决了模块依赖的问题。

    • CMD
      CMD则是seaJS倡导的一种解决模块之间相互依赖规范,推崇依赖就近；在seaJS中一个脚本文件就是一个模块,所有的模块代码写在define的回调函数中，传递三个参数require,exports,module，通过使用 module.exports(exports) 对象向外暴露。

    • commonjs
      每一个文件就是一个模块，拥有自己独立的作用域，变量，以及方法等，对其他的模块都不可见,具有不污染的特性。加载顺序按照代码顺序加载，但只会加载一次，然后缓存起来，同步加载。

  • 如果你写的一个模块导出的是个对象,这个对象里有一个key对应的value是个方法,方法了有this,如果通过结构单独导出这个方法,执行这个方法,this会指向谁
    第一个this指向对象，第二个this指向方法

  • 说下你对symbol的理解,Symbol.for的缺陷
    symbol ：一种数据类型
    Symbol.for 会根据key查找是否已经创建，如果你已经创建key，会直接返回。定义的数据会绑定到全局中

  • es5,bind实现原理
    bind() 函数会创建一个新函数（称为绑定函数），新函数与被调函数（绑定函数的目标函数）具有相同的函数体（在 ECMAScript 5 规范中内置的call属性）。当目标函数被调用时 this 值绑定到 bind() 的第一个参数，该参数不能被重写。绑定函数被调用时，bind() 也接受预设的参数提供给原函数。一个绑定函数也能使用new操作符创建对象：这种行为就像把原函数当成构造器。提供的 this 值被忽略，同时调用时的参数被提供给模拟函数。
